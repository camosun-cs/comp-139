public interface BinaryTree<E>
extends java.lang.Iterable<E>
A sorted collection implemented as a binary search tree. Stored values will be ordered using the tree's current Comparator. Implementations should require a Comparator<? super E> instance as a parameter to all constructors to ensure that add(E) will function correctly for all newly created trees.

    • setComparator

      void setComparator​(java.util.Comparator<? super E> comp)
      Set the current Comparator used for ordering values within this tree. The add(E), min(), max(), and iterator() methods make use of this Comparator to determine the total ordering of all added elements. When the current Comparator is changed, the resulting order of equivalent elements is implementation defined (that is, previous orderings are not necessarily stable).
      Parameters:
      comp - new current Comparator instance to use for future ordering.

    • iterator

      java.util.Iterator<E> iterator()
      Returns an inorder iterator over the elements in this tree. The iterator created by this method is fail-safe; that is, it captures a snapshot of the current tree contents when initialized and will not be affected by later changes to the tree. The Iterator.remove() method must be supported, functioning equivalently to calling remove(E) on the underlying tree. Elements are returned in the order specified by the current Comparator at time of construction (using an "inorder" tree traversal).
